# 🍽️ 맛집맵 (matjipmap.com) 서비스 기획서

> **컨셉**: "돈을 좀 더 쓰더라도, 진짜 맛있는 한 끼"  
> 거지맵이 "어디가 싼가"에 집중했다면, 맛집맵은 **"어디가 진짜 맛있는가"** 에 집중한다.  
> 광고 없이, 사용자 집단지성으로 운영되는 신뢰 기반 맛집 지도.

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 맛집맵 |
| 도메인 | matjipmap.com (또는 맛집맵.com) |
| 핵심 컨셉 | 유저 제보 + 현장 리뷰 검증 기반의 자동 맛집 등급 시스템 |
| 운영 방식 | 집단지성 기반, 관리자 개입 최소화 |
| 앱 설치 | 불필요 (웹 기반, 모바일 최적화) |
| 광고 | 없음 |

### 거지맵 vs 맛집맵 비교

| 구분 | 거지맵 | 맛집맵 |
|------|--------|--------|
| 핵심 기준 | 가격 (7,000원 이하) | 맛 (별점 4.0 이상) |
| 필터 방식 | 가격 필터 (5,000~10,000원) | 음식 카테고리 + 별점 필터 |
| 검증 방식 | 가성비 ↑/↓ 버튼 | 별점 + 현장 리뷰 텍스트 |
| 등재 기준 | 가격 기준 충족 | 현장 리뷰 1개 이상 + 평균 별점 4.0↑ |
| 탈락 조건 | 없음 | 평균 별점 4.0 미만 |

## 2. 핵심 기능 명세

### 2-1. 지도 기반 UI (메인 화면)

- **현재 위치 기반** 주변 맛집 핀(Pin) 표시
- 핀 색상으로 상태 구분:
  - 🟡 **노란색 핀** → "제보됨 (미검증)" 상태
  - 🔴 **빨간색 핀** → "공식 맛집" 상태 (현장 리뷰 1개 이상 + 별점 4.0↑)
  - ⚫ **회색 핀** → 탈락 (별점 4.0 미만, 지도에서 기본 숨김)
- 지도 확대/축소 시 핀 클러스터링
- 핀 클릭 시 요약 카드 표시 (가게명, 대표 메뉴, 별점, 상태 배지)

### 2-2. 가게 상세 페이지

각 가게 핀 클릭 → 상세 페이지 진입

**표시 정보:**
- 가게명, 주소, 전화번호
- 음식 카테고리 (한식 / 일식 / 중식 / 양식 / 분식 / 기타)
- 대표 메뉴 + 가격대 (예: 갈비탕 15,000원)
- **상태 배지** (하단 섹션 3 참고)
- 평균 별점 (★ 표시, 소수점 1자리)
- 총 리뷰 수
- 리뷰 목록 (최신순 / 별점 높은 순 정렬)
- 사진 업로드 가능 (리뷰 작성 시)
- 제보자 정보 (닉네임 또는 익명)

### 2-3. 제보 기능

누구나 맛집을 제보할 수 있음 (회원가입 불필요, 닉네임 선택)

**제보 입력 항목:**
```
- 가게명 (필수)
- 주소 (지도에서 핀 찍기 또는 직접 입력)
- 음식 카테고리 (필수)
- 대표 메뉴 + 가격 (필수, 최소 1개)
- 제보 사유 / 한 줄 추천 (선택, 최대 100자)
- 사진 첨부 (선택)
```

제보 즉시 → **"미검증 맛집"** 상태로 지도에 등록됨

## 3. 맛집 상태 시스템 (핵심 로직)

이 서비스의 핵심은 **관리자가 개입하지 않아도 자동으로 맛집이 관리되는 구조**다.

### 상태 종류

```
[미검증] ──(현장 리뷰 1개 이상 + 별점 4.0↑)──▶ [공식 맛집]
                                                        │
                                               (평균 별점 < 4.0)
                                                        │
                                                        ▼
                                                    [탈락]
                                                        │
                                          (탈락 후 추가 제보 3건 이상)
                                                        │
                                                        ▼
                                               [재등극 심사 중]
                                                        │
                                          (현장 리뷰 1개 이상 + 별점 4.0↑)
                                                        │
                                                        ▼
                                                [공식 맛집 복귀]
```

### 상태별 상세 조건

| 상태 | 조건 | 지도 표시 | 배지 |
|------|------|-----------|------|
| 미검증 | 제보 직후, 현장 리뷰 0개 | ✅ 노란 핀으로 표시 | 🏷️ "아직 검증 전이에요" |
| 공식 맛집 | 현장 리뷰 ≥ 1개 AND 평균 별점 ≥ 4.0 | ✅ 빨간 핀으로 표시 | 🏅 "맛집 인증" |
| 탈락 | 평균 별점 < 4.0 | ❌ 기본 숨김 (필터로 볼 수 있음) | ☁️ "현재 맛집 제외" |
| 재등극 심사 중 | 탈락 후 추가 제보 ≥ 3건 | ✅ 노란 핀으로 재표시 | 🔄 "재검증 중" |

### 상태 전환 규칙 (자동 처리)

1. **미검증 → 공식 맛집**  
   - 조건: 현장 리뷰가 1개라도 등록되고, 해당 시점 평균 별점 ≥ 4.0  
   - 처리: 자동으로 핀 색상 변경, 배지 변경

2. **공식 맛집 → 탈락**  
   - 조건: 누적 리뷰의 평균 별점이 4.0 미만으로 떨어짐  
   - 처리: 지도에서 자동 숨김, 탈락 배지 부여  
   - 탈락 시 가게 상세 페이지는 URL 직접 접근 시 열람 가능 (히스토리 보존)

3. **탈락 → 재등극 심사 중**  
   - 조건: 탈락 이후 새로운 제보가 3건 이상 누적  
   - 처리: 지도에 재표시 (노란 핀), "재검증 중" 배지

4. **재등극 심사 중 → 공식 맛집 복귀**  
   - 조건: 재등극 심사 중 상태에서 현장 리뷰 1개 이상 + 평균 별점 ≥ 4.0  
   - 처리: 공식 맛집으로 복귀

> 💡 **설계 의도**: 관리자가 직접 심사하지 않아도, 유저들의 리뷰와 제보만으로 맛집 목록이 자동 갱신된다.

## 4. 별점 & 리뷰 시스템

### 리뷰 작성 조건

- "현장 리뷰"와 "일반 리뷰"를 구분함
- **현장 리뷰**: 위치 정보(GPS)가 해당 가게 반경 200m 이내에서 작성된 리뷰  
  → 상태 전환(미검증 → 맛집)에 유효한 리뷰로 인정
- **일반 리뷰**: 위치 무관하게 작성 가능  
  → 별점 평균 계산에 포함되나, 상태 전환 트리거 역할은 하지 않음

### 리뷰 입력 항목

```
- 별점 (1~5점, 0.5점 단위)
- 방문 날짜 (월/년 수준)
- 리뷰 텍스트 (최대 500자)
- 사진 첨부 (선택, 최대 3장)
- 재방문 의향 (예/아니오)
- 닉네임 (또는 익명)
```

### 별점 계산 방식

- 모든 리뷰의 단순 평균
- 리뷰 수가 0개인 경우 별점 없음 (미검증 상태 유지)
- 탈락 판정은 **리뷰 수 ≥ 3개**인 경우에만 발동  
  (리뷰 1~2개로 별점이 4.0 미만이라도 탈락하지 않음, 추가 리뷰 대기)

### 리뷰 신고 기능

- 허위 리뷰 / 광고성 리뷰 신고 가능
- 동일 IP에서 단기간 다수 리뷰 작성 시 자동 플래그 처리
- 신고 누적 3회 → 관리자 검토 대기 상태로 전환

## 7. 관리자 기능 (최소화 설계)

자동화로 처리되지 않는 예외 상황만 관리자가 개입

| 기능 | 설명 |
|------|------|
| 신고 리뷰 처리 | 신고 3회 이상 누적된 리뷰 검토 및 삭제 |
| 중복 가게 통합 | 같은 가게가 두 번 등록된 경우 병합 |
| 폐업 처리 | 폐업 신고 다수 접수 시 가게 비활성화 |
| 공지 등록 | 서비스 업데이트 공지 |
| 어뷰징 계정 차단 | 동일 IP 반복 도배 차단 |

> ✅ 별점에 따른 맛집 등극/탈락/재등극은 전부 자동 처리. 관리자가 손댈 필요 없음.

## 8. 기술 스택 제안

### 프론트엔드
- **React** (또는 Next.js - SEO 대응)
- 지도: **카카오맵 API** 또는 **네이버 지도 API** (국내 주소 정확도 우수)
- 모바일 PWA 지원 (홈 화면 추가 시 앱처럼 동작)

### 백엔드
- **Node.js (Express)** 또는 **Python (FastAPI)**
- 상태 전환 로직은 리뷰 저장 시마다 자동 트리거

### DB
- **PostgreSQL** (지리 데이터: PostGIS 확장)
- 가게 위치 좌표 기반 반경 검색 지원

### 인프라
- Vercel (프론트) + Railway / Supabase (백엔드/DB) → 초기 비용 최소화
- 이미지 저장: Cloudflare R2 또는 AWS S3

## 9. 데이터 모델 (간략)

### 가게 (restaurant)
```
id, name, address, lat, lng, category,
status (pending | verified | eliminated | reviewing),
report_count_after_elimination,
created_at, updated_at
```

### 리뷰 (review)
```
id, restaurant_id, nickname,
rating (0.5~5.0), text, visited_at,
is_onsite (boolean), photos[],
revisit_intention (boolean),
flagged, created_at
```

### 제보 (report)
```
id, restaurant_id (nullable - 신규 제보 시 null),
nickname, menu_name, price,
description, photo, created_at
```

## 10. 런칭 전략

### 초기 데이터 시딩
- 개발자가 직접 서울/수도권 유명 맛집 50~100곳 등록 (공식 맛집 상태로 시작)
- 제보 유도 SNS 마케팅 (인스타그램, 스레드 등)

### 바이럴 포인트
- "내가 발굴한 맛집이 공식 맛집으로 등극!" → 제보자에게 알림 발송
- "우리 동네 숨은 맛집 찾기" 챌린지

### 운영 원칙
- 광고 없음
- 업주 직접 등록 불가 (제3자 제보만 허용)
- 모든 상태 변경 이력 투명하게 공개
